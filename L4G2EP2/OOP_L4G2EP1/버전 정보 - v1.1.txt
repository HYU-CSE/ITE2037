L4G2EP1 프로젝트 버전 정보

Version 1.1

이 Java project는 이번 L4G2EP1 프로젝트의 최종 버전입니다.

Version 0.9에서 달라진 점:

	-	플레이어 테스트에 사용할 수 있는 새로운 mode가 추가되었습니다.
		mode 값을 4로 설정하고 프로젝트를 실행하면 창이 하나 뜨며 게임 한 판 또는 100판을 진행할 수 있습니다.
		이를 통해 원하는 게임 번호를 입력하고 버튼을 누르며 게임 진행 상황을 시각적으로 확인해 보거나
		임의의 여러 게임을 진행하며 자신의 플레이어가 높은 / 낮은 점수를 받는 게임 번호를 찾아 분석해 볼 수 있습니다. 
		(테스트용 Presenter는 Start 버튼을 누를 때마다 새 강의실을 생성하므로 한 번 켠 다음 여러 게임을 진행해 볼 수 있습니다)
		
			>	창 좌측 하단에는 특정 게임 번호를 입력하고 한 게임을 진행할 수 있는 버튼들이 모여 있습니다.
				텍스트박스에 원하는 번호를 입력한 다음 Start 버튼을 누르면 게임이 시작되며
				아래의 Advance Turn 버튼을 눌러 다음 턴으로 넘어갈 수 있습니다.
				(게임 번호를 입력하지 않았거나 정수가 아닌 값을 입력한 경우 '임의의 번호'를 사용하여 게임을 진행합니다)
				
			>	한 턴 한 턴 실행을 관두고 최종 결과를 보려면 게임 도중 Skip to end 버튼을 누르세요.
			
			>	강의실 화면 아래에 있는 체크박스를 통해
				관찰할 범위를 '강의실 전체'와 '내 플레이어 시점' 중 선택할 수 있습니다.
				
	(중요)	>	또한, 체크박스를 눌러 포커스를 잡아 놓았을 때(체크박스가 파랗게 변하고 네모가 쳐져 있을 때) 엔터 키를 누르면
				Advance Turn 버튼을 누른 것과 동일하게 매 턴을 진행시킬 수 있습니다.
				매 게임마다 121번 클릭하는 것은 고통스러우니 꼭 이 기능을 이용하세요.
				
			>>	창 우측에는 특정 문자열을 seed로 사용하여 임의의 100판을 자동 진행할 수 있는 기능이 마련되어 있습니다.
				텍스트박스에 아무 문자열이나 입력한 다음 Run 버튼을 누르면 자동 테스트가 시작되며
				각 게임의 결과(내 총점 및 순위)가 아래에 나열됩니다.
				자신이 높은 / 낮은 점수를 받는 게임 번호가 무엇인지 확인한 다음
				창 왼쪽의 '한 판 진행' 기능을 사용하여 해당 게임을 재연해 보면서
				점수 차이의 원인을 분석해 보면 되겠습니다.
				
			>>>	테스트용 Presenter를 사용하더라도 플레이어가 일으키는 오류들은 Eclipse의 콘솔 창에 계속 표시됩니다.
			

	-	이제 유효하지 않은 의사 결정이나 런타임 오류로 패널티를 받게 된 경우
	 	콘솔 창에 해당 패널티를 받은 게임 번호 및 턴 번호를 같이 출력해 줍니다.
	 	
	-	이제 mode 값을 2로 설정하고 임의의 10,000판을 진행하면
		콘솔 창에 해당 게임들 중 내가 가장 높은 점수를 기록한 게임의 번호를 같이 출력해 줍니다.
		
	-	프로젝트 폴더 안에 정규 게임에 사용할 10000numbers.txt를 담아 두었습니다.
		mode 값을 3으로 설정하고 실행하면 정규 게임과 같은 게임 번호 10,000개를 사용하여 게임을 진행하고 최종 평점을 확인해 볼 수 있습니다.
		
	-	그 외에,
		게임 규칙이나 여러분 코드에 영향을 주지 않는 수준에서 자잘한 성능 개선을 수행했습니다.
		
		
Version 0.10에서 달라진 점:

	-	경연용 mode가 추가되었습니다.
		mode 값을 5로 설정하고 프로젝트를 실행하면 창이 하나 뜨며 게임 한 판 또는 정규 게임(10,000판)을 진행할 수 있습니다.
		이 mode는 여러분이 제출한 모든 플레이어 클래스를 등록하여 수업시간에 실행하고 결과 확인 및 시상을 수행하기 위해 만들어졌습니다.  

			'One Game' 부분

			>	'Focus on specific player' 체크박스를 체크한 다음
				바로 아래에 있는 리스트에서 특정 플레이어를 선택하면
				해당 플레이어의 관점에서 강의실 및 게임 정보를 볼 수 있습니다.
				체크박스를 해제하면 강의실 전체를 볼 수 있습니다.
		
			>	'Game number:' 아래에 있는 	텍스트박스에 원하는 번호를 입력한 다음
				'Start one game' 버튼을 누르면 게임이 시작되며
				아래의 Advance Turn 버튼을 눌러 다음 턴으로 넘어갈 수 있습니다.
				(게임 번호를 입력하지 않았거나 정수가 아닌 값을 입력한 경우 '임의의 번호'를 사용하여 게임을 진행합니다)
				
			>	한 턴 한 턴 실행을 관두고 최종 결과를 보려면 게임 도중 Skip to end 버튼을 누르세요.
			
	(중요)	>	또한, 체크박스, 플레이어 리스트, 'One game' 라디오 버튼을 눌러 포커스를 잡아 놓았을 때(파랗게 변하고 네모가 쳐져 있을 때) 엔터 키를 누르면
				Advance Turn 버튼을 누른 것과 동일하게 매 턴을 진행시킬 수 있습니다.
				매 게임마다 121번 클릭하는 것은 고통스러우니 꼭 이 기능을 이용하세요.
				
			'Official Game' 부분
				
			>>	'Run Official Game' 버튼을 누르면
				미리 공지된 10,000개의 게임 번호를 사용하여 정규 게임을 진행하며
				진행 결과를 매 게임이 끝날 때마다 화면에 보여줍니다.
				
			'View Selection' 부분
				
			>>>	두 라디오버튼 중 하나를 눌러 게임 하나에 대한 정보 또는 정규 게임 진행 정보를 선택하여 볼 수 있습니다.
			
				
	-	(핫픽스 누적 적용)플레이어에게 '현재 점수 목록'이 전달되지 않던 오류가 수정되었습니다.
	
	-	플레이어의 잘못된 의사 결정 / 런타임 오류 횟수가 이제 정상적으로 집계됩니다.
	
	
Version 0.11에서 달라진 점:

	-	최종 배포용 mode가 추가되었습니다.
		mode 값을 6으로 설정하고 프로젝트를 실행하면 창이 하나 뜨며 직접 조작 가능한 플레이어를 통해 임의의 게임 한 판을 진행할 수 있습니다.
		이 mode는 학생 여러분이 무료한 시험 기간에 논리력을 일정 수준 향상시키며 동시에 얼마 남지 않은 시간을 더욱 빠르게 소모할 수 있도록 돕습니다.
		
			>	키보드와 마우스를 통해 플레이어 '나'를 조작할 수 있습니다.
				방향 키 또는 마우스 포인터를 통해 커서를 이동시키며
				엔터 키, 스페이스바, 또는 마우스 클릭을 통해 해당 칸을 선택할 수 있습니다.
				
			>	'나'의 위치는 각 상태 아이콘과 같은 색의 화살표로 표시됩니다.
				
			>	선택할 수 없는 칸을 가리키고 있는 경우 커서에 'X' 표시가 추가로 나타납니다.
				아무 것도 하지 않는 대기 의사 결정을 내릴 때를 제외하면 항상 'X' 표시가 나타나지 않는 유효한 칸을 선택해야 합니다.
				
			>	'직접 감염' 선택은 다음과 같이 할 수 있습니다:
			
				- 직접 감염 수락: 엔터 키 또는 마우스 왼쪽 버튼 클릭
				
				- 직접 감염 거절: 스페이스 바 또는 마우스 '왼쪽 버튼 말고 다른 버튼' 클릭
				
				이번 턴에 직접 감염 수락 여부를 선택해야 하는 경우 '내 정보' 창 하단에 해당 내용을 알려줍니다.
				(하지만 1턴째를 제외하면 크게 신경쓸 필요는 없겠군요)
